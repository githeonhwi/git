# YouTube Thumbnail Copy Generator

유튜브 영상 링크를 받아 **썸네일 문구 이미지 3개**를 만든다.
3개는 서브카피의 "근거 유형"이 다르며, 모두 **흰 배경 + 검은 글씨**로 출력한다.

## 입력값

- **YouTube 링크** (일부공개/공개 모두 가능): 필수

## 실행 절차

### 1단계: 영상 내용 파악

`WebFetch`로 영상 페이지를 열어 제목·설명·자막(있으면)을 읽는다.
→ 핵심 주제 1줄, 콘텐츠명(드라마·영화·프로그램명), 핵심 키워드 3~5개 추출

### 2단계: 웹 리서치

`WebSearch`로 동일 주제 레퍼런스를 수집한다. 우선순위:

1. **유튜브** — 조회수 높은 영상 제목 패턴 (`site:youtube.com [키워드]`)
2. **뉴스/웹매거진** — 최신 이슈·화제 앵글
3. **X(트위터)** — 실시간 반응, 밈·유행어 (`[키워드] site:x.com`)
4. **다음 카페** — 실제 사용자 반응·관심사·자주 쓰는 표현 (`site:cafe.daum.net [키워드]`)

리서치에서 반드시 수집할 항목:
- **검증 가능한 수치 지표**: 수상 내역(에미상, 골든글로브 등), 평점(IMDb, 로튼토마토 등), 조회수, 시청률 등
- 시청자가 "이건 봐야 해"라고 납득할 근거

### 3단계: 문구 작성

아래 구조와 규칙으로 3가지 문구를 작성한다.

#### 문구 구조

| 위치 | 명칭 | 역할 | 글자 수 제한 |
|------|------|------|-------------|
| 윗줄 | 서브카피 | **"왜 봐야 하는가"의 근거** — 수치 지표, 장르, 콘텐츠명 조합 | **띄어쓰기 제외 12자 이하** |
| 아랫줄 | 헤드카피 | **시청자 행동·감정 직접 자극** — 강한 단어로 클릭 유발 | **띄어쓰기 제외 10자 이하** |

#### 필수 규칙

1. **콘텐츠명 명시**: 서브카피 또는 헤드카피 중 최소 하나에 콘텐츠명을 `<콘텐츠명>` 형식으로 포함
2. **헤드카피 강도**: 강한 감정·행동 유발 단어 사용. 단순 설명("피날레", "결말", "리뷰")으로 끝내지 않는다
3. **3안은 서브카피의 근거 유형을 달리한다**:
   - **1안 (수치형)**: 수상·평점 등 검증 지표를 서브카피에 배치
   - **2안 (장르+작품명형)**: 장르 설명 + 작품명을 서브카피에 배치
   - **3안 (감성/행동형)**: "인생", "정주행", "역대급" 등 감성 수식어를 서브카피에 배치
4. **헤드카피 3개는 서로 다른 클릭 심리를 자극해야 한다**. 아래 3가지 메커니즘에서 하나씩 배분:

   | 유형 | 클릭 심리 | 헤드카피 방향 | 예시 단어 |
   |------|---------|------------|---------|
   | **사회적 증거형** | "검증됐으니 나도 봐야지" (FOMO) | 권위·순위·대세를 결론으로 | 끝판왕, 레전드, 역대 1위, 요즘 다 본다 |
   | **궁금증 갭형** | "어떻게? 왜? 그게 가능해?" (호기심) | 예상 밖 결합·반전을 암시 | 이게 됩니까, 실화냐, 대체 왜, 이럴 수가 |
   | **자기투영형** | "저게 내 얘기, 나도 해봐야지" (정체성) | 시청자 행동·경험으로 연결 | 인생작, 정주행, 입문 필수, 10분 요약, 완벽 가이드 |

   작성 후 반드시 체크: **3개 헤드카피가 같은 메커니즘에 몰려있으면 재작성한다.**

#### 좋은 예시 패턴

```
1안: "에미상 21관왕 <더 베어>"        /  "요리 드라마 끝판왕"      ← 사회적 증거형
2안: "요리 드라마인데 왜 이래 무서워"  /  "<더 베어> 이게 됩니까"   ← 궁금증 갭형
3안: "인생 요리 드라마 <더 베어>"     /  "정주행 완벽 가이드"       ← 자기투영형
```

### 4단계: 이미지 생성

```bash
python ~/.claude/skills/youtube-thumbnail/scripts/make_thumbnail.py \
  --copies "서브카피1|헤드카피1" "서브카피2|헤드카피2" "서브카피3|헤드카피3" \
  --labels "1안설명" "2안설명" "3안설명" \
  --out-dir "./out"
```

`thumbnail_1.png`, `thumbnail_2.png`, `thumbnail_3.png` 3개가 생성된다.

## 출력 형식

이미지 생성 전 텍스트로 먼저 보여준다:

```
📌 핵심 주제: [한 줄 요약]
🔍 리서치 근거 지표: [수치·수상 등 수집된 검증 지표]

1번 이미지 (사회적 증거형)
  서브카피: [문구]  ([글자수]자)
  헤드카피: [문구]  ([글자수]자)

2번 이미지 (궁금증 갭형)
  서브카피: [문구]  ([글자수]자)
  헤드카피: [문구]  ([글자수]자)

3번 이미지 (자기투영형)
  서브카피: [문구]  ([글자수]자)
  헤드카피: [문구]  ([글자수]자)
```

이후 스크립트를 실행해 PNG 3개를 생성하고 전달한다.

## 품질 기준

- 글자 수는 띄어쓰기를 제외한 실제 글자 수로 계산한다. 초과하면 반드시 다듬어 재작성한다.
- 리서치에서 수치 지표를 찾지 못하면 검색어를 바꿔 재시도한다.
- 3안의 서브카피 유형이 겹치면 가장 약한 것을 다시 써서 방향을 벌린다.
- 헤드카피 3개가 같은 심리 메커니즘(사회적 증거형/궁금증 갭형/자기투영형)에 몰리면 반드시 재작성한다.
- "10분 요약", "n분 정리" 등 시간 기반 문구는 실제 영상 길이를 확인한 후에만 사용한다. 확인 전에는 쓰지 않는다.
- 콘텐츠명(드라마·애니·영화·출연자 등 영상의 핵심 키워드)은 3개 이미지 모두에 반드시 포함한다. 서브카피 또는 헤드카피 어디든 상관없다.
- 광고성·과장 표현("무조건 대박", "역대 최고")은 피한다.
- 한국어 유튜브 시청자 톤 유지. 영어 직역투 금지.
